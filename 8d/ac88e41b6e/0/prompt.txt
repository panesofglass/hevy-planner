## Finding 3: SSE error response pattern repeated 12+ times with no helper

**File:** `/Users/ryanr/Code/hevy-planner/src/index.ts` lines 307, 320, 560, 719, 731, 744, 772, 878, 915, 1066, 1101, 1135

**Details:** The pattern `sseResponse(patchElements(\`<div class="card"><p style="color:var(--orange)">${msg}</p></div>\`, { selector, mode }))` appears 12 times across the file. Each instance builds the same orange-text error card HTML, varying only the message, selector, and mode.

**Recommendation:** Create a helper function:
```typescript
function sseErrorCard(msg: string, selector = "#content", mode: "inner" | "prepend" = "prepend"): Response {
  return sseResponse(
    patchElements(
      `<div class="card"><p style="color:var(--orange)">${escapeHtml(msg)}</p></div>`,
      { selector, mode }
    )
  );
}
```
This reduces each call site to a single line and centralizes the error card styling. Note that some current call sites forget to escape `msg` (the static-string ones are safe, but the pattern is fragile) -- the helper would enforce escaping consistently.

## Finding 7: `deleteWebhookSubscription` bypasses the shared `this.request()` method

**File:** `/Users/ryanr/Code/hevy-planner/src/hevy/client.ts` lines 257-267

**Details:** Every other method in `HevyClient` uses the private `this.request()` helper (line 54), which handles header injection, rate-limit detection (`429`), error formatting, and JSON parsing. `deleteWebhookSubscription` manually calls `fetch()` with hand-rolled headers and error handling. **Recommendation:** Use `this.request<void>("/webhook_subscriptions/${webhookId}", { method: "DELETE" })`. The `request` method will need a small tweak to handle void/empty responses (204), but that is better than maintaining a parallel fetch path.
